# 낙찰/판매가 추정 웹 MVP

## 실행
1. PowerShell에서 작업 폴더로 이동
2. 실행

```powershell
py -3 app.py
```

3. 브라우저 열기

```text
http://127.0.0.1:8080
```

## 저장 구조
- DB 파일: `data.db`
- 옵션: `car_options`
- 경매결과 이력: `auction_results`
- 수동 가격 업데이트 이력: `price_updates`

## 핵심 기능
- 차량정보 입력 후 저장
- 저장 즉시 예상 판매가/입찰가/수익 추정
- 저장된 매물 검색/재조회
- 수동 판매가/입찰가 업데이트
- 경매결과 입력(참여자수/최고가/선택가/내입찰가)
- 경매결과 반영 후 자동 재추정

## 추정 로직(MVP)
- 같은 브랜드/모델/트림 저장 매물 우선 비교
- 수동가격/경매최고가/기존추정가를 앵커로 가중 평균
- 주행거리/연식 보정
- 옵션 보정(저장 데이터에서 학습 가능한 경우만 반영)
- 사고/진단/관리상태 패널티 반영
- 예상 수익 = 예상 판매가 - 예상 입찰가 - 고정비(기본 120만원)

## 다음 개선 추천

## 엔카 자동 가져오기
- 상단 '엔카 자동 가져오기'에 URL 또는 carId 입력
- 차량정보/옵션/진단요약/유사시세 자동 반영 후 저장
- 첫 데이터가 적어도 엔카 시세 중앙값을 초기 기준가로 사용

- 텍스트 원문 분석: 차량정보 블록 붙여넣기 후 자동 파싱/시세조회/저장
